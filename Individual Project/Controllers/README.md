
# Image Processor - gRPC Image Processing Service

A stateless image processing service based on gRPC. The client sends image byte streams + operation list, the server executes the pipeline in sequence and returns the processed image. The image only exists in memory during processing and is not persistently stored.

## Project Structure

```
image-processor/
├── proto/
│   ├── image_processing.proto        # protobuf definition of gRPC service and messages
│   ├── image_processing_pb2.py       # message classes generated by protoc (auto-generated)
│   └── image_processing_pb2_grpc.py  # service classes generated by protoc (auto-generated)
├── server/
│   ├── main.py            # service entry point, configures port, thread pool, message size limits
│   ├── grpc_server.py     # gRPC service implementation, handles request validation, image encoding/decoding, calls pipeline
│   ├── pipeline.py        # pipeline executor, executes operations in order, stops immediately on failure
│   └── operations.py      # 5 image operation functions (rotate/resize/flip/grayscale/thumbnail)
├── client/
│   └── client.py          # gRPC client, encapsulates ImageProcessorClient class
├── tests/
│   ├── test_operations.py  # 15 unit tests, cover normal and error cases for each operation
│   └── test_integration.py # 9 integration tests, full gRPC client-server interaction
├── start.sh               # server startup script
└── requirements.txt        # Python dependencies
```

## System Architecture

```
┌────────┐    gRPC (protobuf)    ┌──────────────────────────────────┐
│ Client │ ──────────────────▶  │           gRPC Server             │
│        │  image_data (bytes)   │                                  │
│        │  operations []        │  1. decode image (bytes → PIL)   │
│        │  generate_thumbnail   │  2. validate operations          │
│        │                       │  3. pipeline: loop operations     │
│        │ ◀──────────────────  │     → apply transformation       │
│        │  processed_image      │     → error? stop immediately    │
│        │  thumbnail (optional) │  4. encode image (PIL → bytes)   │
│        │  error_message        │  5. generate thumbnail (optional)│
└────────┘                       │  6. return result                │
                                 └──────────────────────────────────┘
```

**Core Feature**: Stateless, images are not stored and only exist in memory during processing.

## Supported Operations

| Operation   | Enum Value      | Parameters                               | Description                                                             |
| ----------- | --------------- | ---------------------------------------- | ----------------------------------------------------------------------- |
| `rotate`    | `ROTATE (0)`    | `angle`: float (degrees)                 | Rotate the image, expand=True keeps full content, white background fill |
| `resize`    | `RESIZE (1)`    | `width`: int, `height`: int              | Resize the image using LANCZOS high-quality sampling                    |
| `flip`      | `FLIP (2)`      | `direction`: HORIZONTAL(0) / VERTICAL(1) | Flip horizontally or vertically                                         |
| `grayscale` | `GRAYSCALE (3)` | None                                     | Convert RGB to grayscale                                                |
| `thumbnail` | —               | `max_width`: int, `max_height`: int      | Generate a thumbnail (keep aspect ratio), returned as optional output   |


**Each operation is a pure function**：receives a PIL Image → returns a new PIL Image, without modifying the original image.

```bash

# 2. Compile protobuf (generate Python gRPC code)

服务配置：

- 线程池：10 个 worker
- 最大消息大小：50MB（收发均限制）
- 最大图片大小：20MB（服务端验证）

# Usage: python -m client.client <image_path> [server_address] [port]

python -m client.client photo.png

# connect to remote service
python -m client.client photo.png 42.51.39.84 50051

Command line mode will execute the default demo pipeline (rotate 90° → resize 640x480 → flip horizontal → grayscale), and output processed_output.png and thumbnail_output.png.

### Python Code Usage

```python
from client.client import ImageProcessorClient

# 1.  Create client
client = ImageProcessorClient(host="42.51.39.84", port=50051)

# 2. Define operation list (executed in order)
operations = [
    {"type": "rotate", "angle": 45},           # rotate 45 degrees
    {"type": "resize", "width": 800, "height": 600},  # resize to 800x600
    {"type": "flip", "direction": "horizontal"},       # flip horizontally
    {"type": "grayscale"},                             # convert to grayscale
]

# 3. Send processing request
result = client.process_image(
    image_path="input.png",          # local image path
    operations=operations,           # operation list
    generate_thumbnail=True,         # whether to generate thumbnail
    thumbnail_size=(128, 128),       # maximum thumbnail size
)

# 4. Handle result
if result["success"]:
    # save processed image
    with open("output.png", "wb") as f:
        f.write(result["image"])

    # save thumbnail (if requested)
    if result["thumbnail"]:
        with open("thumb.png", "wb") as f:
            f.write(result["thumbnail"])

    print(f"MIME type: {result['mime_type']}")
else:
    print(f"Processing failed: {result['error']}")

# 5. Close connection
client.close()
```

### Operation Parameter Description

```python
# rotate - rotation
{"type": "rotate", "angle": 90}      # rotate 90 degrees counterclockwise
{"type": "rotate", "angle": -45}     # rotate 45 degrees clockwise

# resize - resize
{"type": "resize", "width": 800, "height": 600}  # width and height must be > 0

# flip - flip
{"type": "flip", "direction": "horizontal"}  # horizontal flip (left-right mirror)
{"type": "flip", "direction": "vertical"}    # vertical flip (top-bottom)

# grayscale - grayscale
{"type": "grayscale"}  # no parameters
```
## gRPC Protocol Definition

### Request (ImageRequest)

| Field                | Type               | Description                                  |
| -------------------- | ------------------ | -------------------------------------------- |
| `image_data`         | bytes              | raw image bytes (PNG/JPEG/GIF)               |
| `operations`         | repeated Operation | operation list, executed in order            |
| `generate_thumbnail` | bool               | whether to generate thumbnail                |
| `thumbnail_params`   | ThumbnailParams    | thumbnail parameters (max_width, max_height) |


### Response(ImageResponse)

| Field             | Type   | Description                                  |
| ----------------- | ------ | -------------------------------------------- |
| `success`         | bool   | whether successful                           |
| `processed_image` | bytes  | processed image bytes                        |
| `thumbnail`       | bytes  | thumbnail bytes (returned only if requested) |
| `error_message`   | string | error message (on failure)                   |
| `mime_type`       | string | MIME type of the output image                |

## Module Responsibilities

### server/operations.py
5 pure functions, each takes a PIL Image returns a new Image：
- `rotate(image, angle)` — rotate, expand=True keeps full content
- `resize(image, width, height)` — resize, LANCZOS sampling
- `flip(image, direction)` — flip, 0=horizontal 1=vertical
- `grayscale(image)` — RGB → grayscale
- `generate_thumbnail(image, max_width, max_height)` — thumbnail, keep aspect ratio

### server/pipeline.py
Pipeline executor. Receives a PIL Image + operation list, executes them in order in a for loop. If any step raises an exception, it is wrapped as `PipelineError` and stops immediately, without continuing subsequent operations.


### server/grpc_server.py
gRPC service implementation class ImageProcessorServicer, processing flow:
1.Validate request (image_data not empty, size ≤ 20MB, operations not empty)
2.decode image (bytes → PIL Image)
3.Validate operation parameters (such as width/height > 0 for resize)
4.Call pipeline.execute() to execute operations
5.encode result (PIL Image → bytes)
6.Optionally generate thumbnail
7.Return ImageResponse

### client/client.py
`ImageProcessorClient` class encapsulates gRPC calls. The `process_image()` method receives an image path and a Python dict-formatted operation list, internally converts them into protobuf messages for sending, and returns a dict-formatted result.
